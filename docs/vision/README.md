# Engram Vision Docs

Public-facing narrative docs for explaining Engram beyond the technical
implementation.

Use these as source material for blog posts, launch copy, investor updates,
podcast prep, or website content.

## Included Docs

### [Why We Are Building Engram](./founder-letter.md)

Founder-style explanation of the problem, the thesis, and why memory should be
private, continuous, and personal.

Best for:

- a founder letter
- an "about" page
- an announcement post

### [One Brain Per Person](./one-brain-per-person.md)

Vision article about the product philosophy behind Engram: one private brain,
projects as topology, memory as a living system, not a bigger context window.

Best for:

- a blog post
- a launch essay
- social post threads derived from core arguments

### [The Science Behind Engram](./science-of-engram.md)

Explains the neuroscience and cognitive-science ideas Engram borrows from:
engram traces, cue-dependent recall, consolidation, reconsolidation, forgetting,
and activation-based retrieval.

Best for:

- technical audiences
- a research/approach page
- explaining why Engram is not "just RAG"

### [Federated Policy Intelligence](./federated-policy-intelligence.md)

Explains the long-term differentiator: Engram should not centralize private
memories. It should learn better memory policies from privacy-bounded telemetry
and ship back signed policy packs.

Best for:

- strategy memos
- a future-facing blog post
- explaining the roadmap beyond local memory

## Positioning Notes

Across these docs, the strongest through-line is:

1. AI needs memory, not just larger context windows.
2. Memory should be one private brain per person.
3. Useful memory is selective, layered, and self-organizing.
4. Global learning should improve policies, not absorb private memories.

That is the cleanest narrative Engram has right now.
