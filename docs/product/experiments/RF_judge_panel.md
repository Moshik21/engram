# Recency/Frequency Redesign — Judge Panel (2026-07-18)

Corrected three-lens adversarial panel against the committed design. Scores: cognitive-theory 8/10, engineering-blast-radius 7/10, eval-fitness 6/10. All blockers folded into RECENCY_FREQUENCY_GOAL.md.


## cognitive-theory — 8/10

**Summary:** As a computational cognitive scientist I independently recomputed the §D1 desaturation table and every target holds exactly (1-used=0.067, 5-used=0.225, 50-used=0.296; single access maxes at 0.176 vs the old sigmoid's 0.913, saturation now requiring ~50 confirmed events). The multiplicative router theorem is valid (u∈[0,1] ⇒ boost≤1.30×; sem(X)>sem(Y)/(1+β) is a correct worst-case necessary condition) and provably prevents the 23→2 collapse — gold 0.50 vs person 0.30 can never flip because 0.30 < 0.50/1.25. The tiered recording (surfaced=0/used=0.3/confirmed=1.0/corrected=0.5) is faithful to Anderson-Schooler rational analysis: surfacing is ranker output not environmental need, and the ordering is justifiable (exact magnitudes are honestly deferred to calibration). Episode unification into two separately-gated recencies (environmental creation-time vs behavioral use-time) resolves a genuine category error. Log-compressed frequency is a saturating nonlinearity, but the fix correctly moves the saturation point from first-touch to ~50 events rather than deleting it. Keeping ACT-R for hygiene while u ranks is coherent, not a smell — the consumers need different transforms and one shared formula is the documented root cause. The one real defect: the keystone agent-used signal (the ingestion-time citation scan) IS a self-referential access loop — it can echo the ranker's own surfaced payload back as a w=0.3 'used' event whenever a client observe()s the agent's answer, which by the design's OWN fixed-point argument ('discounting changes the rate, not the fixed point') reopens the loop it claims to have killed and collapses the surfaced<used tiering. The §3 multiplicative bound keeps this from reproducing the catastrophe (it pollutes the within-band tiebreaker, it cannot collapse ranking), so it is a signal-integrity blocker with a cheap, concrete fix (echo guard + dedup + an echo-rig gate), not a fatal one. Strong, genuinely self-critiqued design; buildable once the echo guard lands. Score 8/10.


### Blockers


**B1. The keystone agent-used signal — the ingestion-time citation scan (RF_target_design.md:73; RECENCY_FREQUENCY_REDESIGN.md:106-111) — scans 'the next observed turn' for surfaced entity names via _matches_entity_name and emits a w=0.3 'used' event on a match, with NO guard that the scanned content is not the ranker's own surfaced payload echoed back into the conversation. It is a self-referential access loop, the exact mistake the design claims to have eliminated.**

*Why:* The design kills w_surfaced=0 on the stated principle that 'any nonzero w_surfaced, however heavily discounted, reopens the rich-get-richer loop — discounting changes the rate, not the fixed point' (RECENCY_FREQUENCY_REDESIGN.md:102-104; RF_target_design.md:78). That principle indicts the used scan too: when a client observe()s the agent's ANSWER (the design explicitly targets the answer: 'a surfaced entity the agent then cites in its answer', line 110) — or the full turn — retrieved memories injected into the agent's context are routinely echoed verbatim in the answer, so every surfaced entity matches and accrues a w=0.3 event with zero user judgment involved. By the design's own fixed-point argument, 'used' then degenerates into a discounted surfaced loop and the tiering surfaced(0)<used(0.3) collapses — the used tier is no longer a trusted signal, which is its whole justification. The mechanism's validity rests entirely on WHAT the client chooses to observe(), which Engram does not control (CLAUDE.md's own protocol has agents inject get_context/recall output then observe/remember). NOTE this is signal-integrity pollution, not catastrophe: §3's multiplicative bound (u∈[0,1], boost ≤1.30×) means even a fully-reopened echo loop cannot reproduce the additive 23→2 collapse (M4_1_activation_arm.md:44-49) — it can only pollute the within-band tiebreaker. But the design markets 'used' as the loop-free keystone, and as specified it is not.

*Fix:* Make the used event reflect environmental demand, not ranker output: (a) attribute 'used' only to tokens provably NOT in the surfaced payload — diff the observed content against the ring-buffer summary text and require the entity mention to fall in novel (non-echoed) tokens, or restrict the scan to the user-authored portion of the turn; (b) dedup per (entity_id, conversation) inside a window so a persistently in-context entity cannot accrue runaway used events; (c) add an echo-rig gate to the §6 acceptance set — feed the ranker's own top-k back as the synthetic 'next turn' and assert the used-event count stays ~0, mirroring the G1 no-op proof. Until such a guard exists, hold used at w=0 and ship only confirmed/corrected (explicit user signal, echo-immune).


### Improvements
- u discards ACT-R's spacing/distributed-practice structure: recency uses only Δ_last (the single most-recent access, RF_target_design.md:42) while ACT-R's base-level integrates Σ(t−t_j)^−d over ALL accesses. A well-established memory phenomenon (distributed > massed practice) is lost — 5 accesses over a month and 5 in one minute produce identical n_eff and identical r′ if the last access time matches. Defensible for a cheap, deterministic, O(1) tiebreaker (the design's explicit rationale, line 34), but name the fidelity loss; a 2-3-point recency would recover most of it without the full ln-sum.
- The 2.5x durable-importance ranking multiplier (D6/§5, RF_target_design.md:147) composes MULTIPLICATIVELY with usage but escapes the 'usage never beats semantics' theorem: 2.5x >> the usage cap 1.30x, so a durable-but-semantically-weak item (sem 0.30 → 0.75) beats a non-durable strong item (sem 0.50 → 0.50). That is the same 'a term can dominate semantics' failure class the design fixes for usage, just on the importance axis. It is out of scope (gated by importance_prior_enabled, punted to F6/M3), but the multiplicative COMPOSITION is introduced here — flag explicitly that importance can dominate semantics and either justify it (identity-core surfacing on weak match may be intended) or bring it under an analogous bounded invariant during M3 calibration.

Intentional ('keep old-but-frequent alive'), but staleness has a hard floor with no upper age cutoff. Verify against the §6 calibration that this does not over-protect genuinely abandoned high-frequency items; consider a long-horizon age cutoff above which r′→0.
- The used signal is bracketed by two opposite client-dependent failure modes: observe-the-answer → echo pollution (the blocker); don't-observe-the-answer → ~0 used events = silent-inert, exactly the M4.4 parked concern the design claims to resolve (RECENCY_FREQUENCY_REDESIGN.md:426-427). ## engineering-blast-radius — 7/10

**Summary:** Rigorous, cartography-grounded design with correct arithmetic, a provable kill-switch, and sound core mechanics (one-store-two-views, multiplicative bounded tiebreaker, w_surfaced=0, session-pollution fix, TS kill). Three must-fix blockers before it is buildable, all in the durability/blast-radius layer the task targets. (1) The confirmed-event JSONL journal loses events across the shell/MCP-stdio/brain three-process reality: truncation-on-save is not coordinated with which process's snapshot actually incorporated the event, so MCP-stdio-authored confirmed signals are dropped when the shell shuts down and truncates — data loss for the exact tier the journal protects. (2) D3 removes activation from ranking in only two of the seven ranking-path readers the cartography enumerates; spreading temporal_mode seed energy (spreading.py:42-54, verified) still reads compute_activation and reproduces a slice of the 23->2 collapse on TEMPORAL queries — and the design's own 'populated==empty store' regression gate cannot pass until these orphans are neutralized, yet they aren't listed. This is precisely the 'three switches, not one' orphan the cartography warned would be the next silent-inert bug. (3) Environmental mention-frequency (P5 ingestion) is silently mapped to w=0, so ranking-frequency depends entirely on the unproven citation scan whose keystone assumption (agents observe their own answers) is unvalidated and whose live event volume is never gated — the project's recurring computed-but-silently-inert failure mode. Fix the journal ordering, enumerate/gate all activation readers, and either give ingestion mentions a loop-free frequency tier or add a live-volume silent-inert guard. Score 7: the thinking is high-quality and most choices should not be relitigated, but the three blockers are real and one is a data-loss race.


### Blockers


**B1. The confirmed-event JSONL journal (D6/§5: 'written through at record time, replayed on load, truncated on successful snapshot save') loses confirmed/corrected events — the exact tier it exists to protect — under the real three-process reality (shell + MCP-stdio + brain).**

*Why:* Replay happens only at load; truncation happens on a per-process snapshot save. A confirmed event written by MCP-stdio while the shell is alive (mcp/server.py:437 makes stdio SKIP its save when serve_process_alive(); :445 skips on mtime change) never enters the shell's in-RAM store. When the shell later saves on clean shutdown and truncates the journal, that event is dropped from the journal, and it was never in the shell's snapshot (different process's RAM), and stdio never persisted it (save skipped). Net: a user's confirmed signal is silently lost. The 'shell paused => single-writer' invariant covers snapshot_to_graph in the mop window, NOT the live journal, which is written by whichever process serves the feedback tool. There is also a TOCTOU: between replay-at-load and truncate-at-save any concurrently-appended line is lost.

*Fix:* Make truncation a fold-then-compact under the SAME ownership protocol as the snapshot: on save, the owning process must (a) re-read and replay the entire current journal into RAM so its snapshot becomes a superset of every journaled confirmed event, (b) write the snapshot, then (c) truncate by rename-and-recreate (capture the old inode), re-scanning any lines appended during the write into a fresh segment. Non-owning processes (stdio while shell alive, brain) append but NEVER truncate. Add a test: event written by a second process is still present after the first process saves+truncates.


**B2. D3's enumerated deletions ('delete the additive w_act*act term' + 'delete the activation-based candidate backfill') are insufficient to remove activation from ranking; the spreading temporal_mode seed, goal priming, temporal bypass, prospective and surprise readers still consume activation and are orphaned. This is the exact 'three separate switches, not one' hazard the cartography named (§4).**

*Why:* Cartography §3 lists SEVEN ranking-path activation readers. D3 addresses two. spreading.py:42-54 (verified) sets threshold=0.0 in temporal_mode and seeds energy=max(act,0.15) where act=compute_activation(state.access_history,...); a saturated wrong-person entity still gets ~0.91 seed energy vs 0.15 for a cold entity — 6x — on exactly the TEMPORAL route that produced the 23->2 collapse. That energy flows into composite_sem_score, which D3 then multiplies by (1+beta*u). So flag-off with a populated store is NOT activation-free, and the design's own regression gate (§3 test 2: 'flag-off recall on a populated activation store is byte-identical to the same recall on an empty store') CANNOT pass until these readers are also neutralized — yet the design does not list them. On the live 8.7k brain (warm store loaded from snapshot; every entity carries >=1 ingestion access via P5), this orphan is live, not theoretical, and reproduces a slice of the collapse on temporal queries.

*Fix:* Enumerate every cartography-§3 ranking-path reader (scorer term scorer.py:84-93 AND the TS twin 241-247, seed energy spreading.py:42-71, activation candidate pool candidate_pool.py:120-148, temporal bypass pipeline.py:536-550, goal priming goals.py:98-107,151, prospective, surprise) and gate/neutralize each under one shared activation_ranking flag, then assert 'populated store == empty store' over a rig that exercises spreading and temporal_mode — not just the scorer. Note the coupling to D7: if F4 chooses FLOOR not KILL, the TS scorer twin keeps the w_act*act term and remains an orphan.


**B3. Environmental mention-frequency (P5 ingestion, extraction/apply.py:847, and P8 bootstrap) is silently mapped to w=0 by the new record_access default tier='surfaced', leaving ranking-frequency dependent ENTIRELY on the unproven citation scan — and there is no live-volume silent-inert guard. This is the project's recurring 'computed-but-silently-inert' bug class.**

*Why:* record_access today is 3-arg and unconditional at ingestion (P5 = 1 access/mention, the loop-FREE environmental frequency signal that Anderson-Schooler need-probability is actually about). The new tier param defaults 'surfaced' (w_ranking=0), so P5/P8 events never enter usage_events and contribute 0 to n_eff => f=0. The M4.1 collapse evidence indicts SURFACING (P1/P9, 108 events on Persons from ranker output), not ingestion — so w=0 for P5 over-excludes based on evidence about a different, loopy signal. Meanwhile n_eff's only live inputs (used/confirmed/corrected) depend on the citation scan, whose keystone assumption — 'the agent's answer reaches Engram via the next observe()' — is unvalidated: agents are not guaranteed to observe() their own answers, and the ring buffer is process-local (same cross-process fragility that killed reconsolidation P6). If citation-scan volume is ~0 in production, f≈0 for everything and the entire usage multiplier is inert. G1-G5 inject synthetic events on the rig; M1.3 is a single hand-crafted trace — neither measures real per-tier event volume on the live brain.

*Fix:* Either (a) give ingestion mentions a loop-free environmental-frequency tier (e.g. w_mentioned ~0.1-0.2, justified because it is user-content-driven not ranker-output-driven), or (b) explicitly argue and document why environmental occurrence is excluded from a system whose stated basis is environmental statistics. Add a hard gate before the usage_ranking flip: measure actual per-tier event volume over a real capture window on live-copy; if used+confirmed+corrected volume is below a floor, the mechanism is declared silent-inert and the flip is blocked.
